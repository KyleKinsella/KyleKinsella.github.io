Learning 1: Git Commands

As I have been building my chess game (engine later on), I have been using Git to keep track of my work. I have used Git before, but I only know a few commands, see below list:

  • git status
  • git diff
  • git add .
  • git commit -m "..."
  • git commit --amend
  • git push
But this is where things make a turn. I am working on this project with a friend from college and when he is working on the project, he has created is own branch for his work. But I was doing my work and pushing to the main branch. But he said you should make your own branch and not push to main.

So, yesterday (June 11th, 2026), I was working away on the project and I had just implemented team colors, a very important part of chess (and a lot of work)! Then I said to myself, I should push this to git, but this time I didnt push to main, I created my own branch, this is the main part of this story / learning.

I have never created my own branch before!

Part of me was like, "just push to main to be safe", because I didnt want to lose my work, that I spent the whole day working on. But, I didnt! I just did it and what happened, happened. So, I went off and did some research and this is what I did and what I learned, see below:
  • git stash
  • git checkout -b Kyles_Work (I needed to create a new branch)
  • git stash pop
  • git push --set-upstream origin Kyles_Work
I also learned about these Git commands:
  • git branch - shows the branch your currently on.
  • git stash list - shows all of the stashes that you have done.

Doing these commands was quite nerve wrecking because I had never used these commands before. But now that I've done it once I will be more confident doing this going forward.
