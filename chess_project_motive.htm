Project Contents

As the project is still very early doors, there is not too much to show off just yet. But the aim of our chess application is to have all of the core features that chess has, within our project. Chess contains the following:

  • A board - 8x8 grid with 64 squares
  • Pieces for each team:
    • 2x Rooks
    • 2x Knights
    • 2x Bishops
    • 1x Queen
    • 1x King
    • 8x Pawns
  • Moving Pieces:
    • legal
    • illegal
  • What do I do ?
    • Checkmate: your king is in danger!
    • Stalemate (Drawing) draw the game and not lose!
    • Winning or Lossing!
  • Special things to do...
    • Castling: swap your king with your rook, to keep your king safe!
    • En passant: a special move only pawns can do!
    • Promotion: our pawn can promote to be either: a queen, rook, bishop or a knight!

Challange 1: Making the Chess Board.

When I started off this project initially I thought making the chess board was going to very straight foward (I was wrong). I was trying to write the best piece of code the first time out - a lot of developers say not to do this, they say "get a brute force solution working first, then refactor it to be better code".

My Brute Force Solution:
  • I manually created a div with 64 other divs within the first div.
  • Then I manually gave each div either a black or white color.
  • Then I again manually gave each square its own piece.
As you can see, this is incredibly bad!

My Optimal Solution:
  • Pass in an empty board and populate it.
  • Pass in the pieces that we want to put onto the board.
  • Pass in the rows and cols to make the 8x8 grid.
  • Checking the value of i.
    • If i is 0 or 7, meaning we are going to place the following pieces: rooks, knights, bishops, queens, kings.
    • If i is 2, 3, 4 or 5, meaning we are going to place "______". This means that this is a free spot, where the pieces can move too.
    • If i is 1 or 6, meaning we are going to place 8x pawns.

Note: this is my current function for making the chess board, I might need to refactor it a bit later on, but it works perfectly for now.

board.jsx

			export function initGameBoard(board, pieces, rows, cols) {
				for (var i = 0; i < rows; i++) {
					for (var j = 0; j < cols; j++) {
				
						if (i === 0 || i === 7) {
							board[i][j] = Utils.pieces[j] + " ";
						}
             
						if (i === 2 || i === 3 || i === 4 || i === 5) {
							board[i][j] = Utils.freeSpace + " ";
						}
                
						if (i === 1 || i === 6) {                
							for (var pawn = 0; pawn < Utils.pieces.length; pawn++) {
								 if (pieces[pawn] === Utils.PAWN) {
									 board[i][j] = Utils.pieces[pawn] + " ";
									 //~ board[i][j] = Utils.freeSpace + " ";
								 }
							}
						}
				}
			}	   
		return board;
	}

Why Go ?

I chose to use Go for this project mainly because I really enjoy coding in Go and Go code is very easy to read and understand. But another reason I chose Go is for the concurreny aspect of Go. We all know the basics of chess and one of the most important things is raw speed for chess and Go's concurrency model is very good, so that is another reason to why I chose Go.

For example:

Chess is a 2d array with Rows and Cols so for a lot of the project i'm going to be iterating twice like this: 

			for i := 0; i < ROWS; i++ { 
				for j := 0; j < COLS; j++ {	
					// process here... 	
				} 	
			}
So, if I do this sequentially this will be quite slow, but if I use Go's builtin Concurrency features, such as: Goroutines, Channels, Waitgroups, etc, I will be able to process the chess game much faster!
